1)Explain History of Java
A) Start (1991):

    - Java began in 1991 with James Gosling and team at Sun Microsystems.
    -They were working on small electronic devices like set-top boxes.
 
    Name Changes:

   -It was first called "Greentalk" and later changed to "Oak."
   -"Oak" was strong, but they had to change it to "Java" in 1995 due to trademark issues.
 
    Why "Java"?

  - "Java" was picked because it's a unique name.
  - Also, James Gosling liked the association with the coffee produced in the Indonesian island called Java.

   Important Points:

  -Java is just a name, not an abbreviation.
  -The first version, JDK 1.0, came out on January 23, 1996.
  
  Growing Use:

  -Java grew and is now used in many things like Windows apps, web apps, and mobile apps.
  -New versions keep adding cool features for internet programming.

3)  Define Byte codes of JAVA and JVM
A) Bytecode in Java:

       -Intermediate code generated by the Java compiler from source code.
      - Platform-independent set of instructions.
 
      Java Virtual Machine (JVM):

      Software emulation of a physical computer.
      -Executes Java bytecode.
      -Translates bytecode into machine-specific instructions.
      Enables Java programs to run on various devices.

4) Explain primitive data types of java
A)  1) byte:

       -Used to represent small integer values efficiently.
       -Commonly used in situations where memory conservation is crucial, like reading from a file or network.
       -Size: 8 bits
       -Range: -128 to 127

     2)short:

       -Represents a larger range of integer values compared to byte.
       -Useful in scenarios where memory space is a concern, but a broader range of values is needed.
       -Size: 16 bits
       -Range: -32,768 to 32,767

     3)int:

       -Most commonly used integer type in Java.
       -Suitable for representing a wide range of integer values in various applications.
       -Size: 32 bits
       -Range: -2^31 to 2^31 - 1

     4)long:

       -Used for situations requiring a larger range of integer values.
       -Often used for representing timestamps or large numerical values.
      -Size: 64 bits
      -Range: -2^63 to 2^63 - 1

     5)float:

       -Represents decimal numbers with single-precision.
       -Suitable for applications where precision is not critical, such as graphics or simulations.
      -Size: 32 bits
      -Represents decimal numbers with single-precision

     6)double:

       -Represents decimal numbers with double-precision.
       -Widely used for applications requiring high precision, such as financial calculations.
       -Size: 64 bits
       -Represents decimal numbers with double-precision
     7)char:

       -Represents a single Unicode character.
       -Used extensively for handling characters in text processing and user interfaces.
       -Size: 16 bits
       -Represents a single character using Unicode encoding
     8)boolean:

       -Represents true or false values.
       -Essential for decision-making and controlling program flow based on conditions.
      -Represents true or false values
